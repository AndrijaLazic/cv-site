This blog continues where my previous DigitalLeather story ended.

After joining as a backend engineer in May 2025, I gradually moved into an AI / Machine Learning Engineer role on FMEA Excellence. That transition made sense to me because I was already close to the product, the users, and the parts of the system where prediction could remove a lot of manual work.

FMEA Excellence has three main AI features: ROI prediction, an AI chatbot, and AI-assisted field completion inside PFMEA tables. Each of them solves a different problem, but they all have the same goal: help people make better decisions without forcing them to become data scientists.

<ContentBlock title="ROI prediction as the core feature">
  The most important part of the product is ROI prediction. It helps managers decide which risks should be solved first based on the money, time, and resources they actually have available.

In this case, managers are usually not technical users. They do not want to inspect raw model output or think about forecasting internals. They want to know where to act, how much an action will cost, when the investment will return, and what will probably happen if they do nothing.

Before this feature, that kind of analysis could take several working days. With the prediction workflow, the user can get the important numbers in one flow: expected ROI, estimated number of days until the investment returns, and a scrap prediction chart for the selected production process.

  <Callout type="info" title="Why this mattered">
    The value was not only in predicting numbers. The bigger value was turning a slow planning process into something managers could use while they were actually making decisions.
  </Callout>
</ContentBlock>

<ContentBlock title="How the prediction workflow works">
  Behind the interface, the system combines multiple ML prediction steps.

The models use scrap data from the selected station to estimate cost, predict the future trend, and calculate ROI. After that, genetic algorithms and linear programming are used to choose the best set of actions under the user's real constraints.

<Section title="What users get from the workflow">
  - ROI estimation for planned actions. - Number of days until the investment is
  expected to return. - Scrap trend prediction for a given production process. -
  Prioritization of actions based on available budget and time.
</Section>

The difficult part was not just building a model that worked on a small example. The real challenge appeared when the number of risks started to grow. More risks meant more calculations, more database work, and longer prediction times.

We solved that by optimizing database queries, parallelizing the calculation flow, and precomputing the parts of the workflow where that was possible. That made the feature much more practical in real product usage.

</ContentBlock>

<ContentBlock title="AI chatbot for FMEA knowledge">
  The second feature is a RAG chatbot that answers questions about the application and about FMEA usage in industry.

The goal was to give users a place where they could ask practical questions without leaving the product. We used LangChain and GPT models as the foundation, with a knowledge base built from internal product material and research work by <a href="https://www.linkedin.com/in/nikola-banduka-phd-67b63ab1/" target="_blank" rel="noreferrer">Dr. Nikola Banduka</a> and <a href="https://www.linkedin.com/in/nikola-komatina-88b6b31a9/" target="_blank" rel="noreferrer">Nikola Komatina</a>, who both completed doctoral work in the field of FMEA analysis.

This feature was useful because FMEA can be complex even for experienced teams. A good assistant can explain concepts, help users understand why a field matters, and guide them through the way the application expects the analysis to be structured.

</ContentBlock>

<ContentBlock title="AI-assisted PFMEA table completion">
  The third feature is similar to a RAG workflow, but it also takes the user's previous data into account.

Instead of only answering questions, the model helps fill fields inside PFMEA tables. It can look at previous analyses, similar processes, and existing user data, then suggest values that make the table easier and faster to complete.

This is where AI becomes less visible but very useful. The user still owns the final decision, but the application removes repetitive typing and helps them start from something relevant instead of a blank table.

  <Callout type="tip" title="Main takeaway">
    The best AI features in this product were not separate demos. They were built into existing workflows, where they could reduce manual work and make decision-making faster.
  </Callout>
</ContentBlock>
